# BSADS — Bee Swarming and Abscondence Detection System

BSADS is an Internet of Things (IoT)–based acoustic and environmental monitoring platform designed to predict and detect swarming or absconding behaviors in beehives. The system integrates acoustic analysis, environmental sensing for hive temperature and humidity, and machine learning models to automatically classify hive state conditions, including **Normal**, **Pre-Swarm**, **Swarm**, **Absconding**, and **Uncertain**.

Its main objective is to help beekeepers make early, data-driven interventions to reduce colony loss and improve apiary management efficiency, while contributing data for apicultural research.

Built with **Laravel 13 + React 19 + Inertia.js + PostgreSQL**.

## Local Development Setup

### Requirements

- PHP 8.3+
- Composer
- Node.js **22.x** (use nvm — see below)
- PostgreSQL client (optional, for direct DB access)

### 1. Clone the repo

```bash
git clone https://github.com/Abinsinguza/beehive-app.git
cd beehive-app
```

### 2. Install dependencies

```bash
composer install
```

For Node, use nvm to ensure the correct version:

```bash
nvm install 22
nvm use 22
npm install
```

### 3. Configure environment

```bash
cp .env.example .env
```

Then open `.env` and fill in your values (see [Environment Variables](#environment-variables) below).

Generate an app key:

```bash
php artisan key:generate
```

### 4. Build frontend assets

```bash
npm run build
```

### 5. Run migrations

> **Important:** The production database already has tables created. Only run migrations against a fresh local database. Never run `migrate:fresh` or `migrate:reset` against a shared/production database.

```bash
php artisan migrate
```

### 6. Start the development server

```bash
php artisan serve
```

Visit `http://localhost:8000`

## Environment Variables

Copy `.env.example` to `.env` and set the following:

```env
APP_NAME=BSADS
APP_ENV=local
APP_KEY=                        # generated by php artisan key:generate
APP_DEBUG=true
APP_URL=http://localhost:8000
FORCE_HTTPS=false                # set true only if a reverse proxy terminates SSL in front of the app

DB_CONNECTION=pgsql
DB_HOST=                        # your PostgreSQL host
DB_PORT=                        # your PostgreSQL port
DB_DATABASE=
DB_USERNAME=
DB_PASSWORD=
DB_SSLMODE=prefer
DB_CONNECT_TIMEOUT=10

SESSION_DRIVER=file
CACHE_STORE=file
QUEUE_CONNECTION=sync
```

> See [DEPLOYMENT.md](DEPLOYMENT.md) for the full environment variable reference and deployment-specific guidance (Docker, generic Linux server).

> **Never commit your real `.env` file.** It contains secrets.

## Key Architecture Notes

- **UUID primary keys** — all tables use descriptive PK names (`hive_id`, `alert_id`, `user_id`, etc.) not generic `id`
- **Soft deletes on hives** — uses `deleted_at` + `is_deleted` columns; records are never hard-deleted
- **Audio inference pipeline** — `audio_sources` → `inference_results` → `alerts` → `advisory_actions`
- **Production schema** — `advisories` is a templates/rules table; per-hive action records live in `advisory_actions`
