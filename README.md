# Beehive Monitoring System

A web-based beehive monitoring platform that uses audio inference to detect hive states (swarming, external noise, healthy, etc.) and alerts beekeepers in real time.

Built with **Laravel 13 + React 19 + Inertia.js + PostgreSQL**, deployed on **Railway**.

---

## Tech Stack

| Layer | Technology |
|---|---|
| Backend | PHP 8.3, Laravel 13, Fortify auth |
| Frontend | React 19, TypeScript, Tailwind CSS 4, Inertia.js v3 |
| Build | Vite 8 (requires Node 20.19+ or 22.12+) |
| Database | PostgreSQL 18 (hosted on Railway) |
| Deployment | Railway (Railpack auto-detected PHP/Laravel) |

## Local Development Setup

### Requirements

- PHP 8.3+
- Composer
- Node.js **22.x** (use nvm — see below)
- PostgreSQL client (optional, for direct DB access)

### 1. Clone the repo

```bash
git clone https://github.com/Abinsinguza/beehive-app.git
cd beehive-app
```

### 2. Install dependencies

```bash
composer install
```

For Node, use nvm to ensure the correct version:

```bash
nvm install 22
nvm use 22
npm install
```

### 3. Configure environment

```bash
cp .env.example .env
```

Then open `.env` and fill in your values (see [Environment Variables](#environment-variables) below).

Generate an app key:

```bash
php artisan key:generate
```

### 4. Build frontend assets

```bash
npm run build
```

### 5. Run migrations

> **Important:** The production database already has tables created. Only run migrations against a fresh local database. Never run `migrate:fresh` or `migrate:reset` against the production Railway database.

```bash
php artisan migrate
```

### 6. Start the development server

```bash
php artisan serve
```

Visit `http://localhost:8000`

## Environment Variables

Copy `.env.example` to `.env` and set the following:

```env
APP_NAME=Laravel
APP_ENV=local
APP_KEY=                        # generated by php artisan key:generate
APP_DEBUG=true
APP_URL=http://localhost:8000

DB_CONNECTION=pgsql
DB_HOST=                        # Railway DB proxy host or IP
DB_PORT=                        # Railway DB proxy port
DB_DATABASE=railway
DB_USERNAME=postgres
DB_PASSWORD=                    # from Railway dashboard → Postgres → Variables
DB_SSLMODE=prefer
DB_CONNECT_TIMEOUT=10

SESSION_DRIVER=file
CACHE_STORE=file
QUEUE_CONNECTION=sync
```

> **Never commit your real `.env` file.** It contains secrets.

## User Roles

| Role | Access |
|---|---|
| `admin` | Full access — manage hives, farmers, alerts, advisories |
| `farmer` | Limited access — view assigned hives and alerts |
| `farmer_pending` | Registered but not yet approved |
| `farmer_revoked` | Access revoked |

---

## Key Architecture Notes

- **UUID primary keys** — all tables use descriptive PK names (`hive_id`, `alert_id`, `user_id`, etc.) not generic `id`
- **Soft deletes on hives** — uses `deleted_at` + `is_deleted` columns; records are never hard-deleted
- **Audio inference pipeline** — `audio_sources` → `inference_results` → `alerts` → `advisory_actions`
- **Production schema** — `advisories` is a templates/rules table; per-hive action records live in `advisory_actions`
