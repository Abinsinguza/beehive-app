# Deployment Guide

This document covers every supported way to install and deploy **BSADS** (Bee Swarming & Abscondence Detection System): local development, Docker, and a generic Linux server.

---

## 1. Requirements

| Requirement | Version |
|---|---|
| PHP | 8.3+ with extensions: `pdo_pgsql`, `mbstring`, `openssl`, `tokenizer`, `xml`, `ctype`, `json`, `bcmath`, `fileinfo` |
| Node.js | 22.x |
| Composer | 2.x |
| PostgreSQL | 17+ (client libraries to connect) |

---

## 2. Environment Variables Reference

Copy `.env.example` to `.env` and fill in real values:

```bash
cp .env.example .env
php artisan key:generate
```

| Variable | Purpose |
|---|---|
| `APP_NAME` | Display name (`BSADS`) |
| `APP_ENV` | `local` / `production` |
| `APP_KEY` | Generated by `php artisan key:generate` — never commit a real one |
| `APP_DEBUG` | `true` locally, `false` in production |
| `APP_URL` | Full base URL the app is served from |
| `FORCE_HTTPS` | Set to `true` **only** when a reverse proxy/edge (e.g. Nginx+Certbot) terminates SSL in front of the app. Leave `false`/unset for plain-HTTP setups (e.g. local Docker testing) — forcing HTTPS URL generation when nothing actually serves TLS breaks every asset load (`net::ERR_SSL_PROTOCOL_ERROR`). |
| `DB_CONNECTION` | `pgsql` |
| `DB_HOST` / `DB_PORT` / `DB_DATABASE` / `DB_USERNAME` / `DB_PASSWORD` | Your PostgreSQL connection details |
| `DB_SSLMODE` | `prefer` (works with most managed Postgres providers) |
| `SESSION_DRIVER` | `file` |
| `CACHE_STORE` | `file` |
| `QUEUE_CONNECTION` | `sync` |

> **Never commit your real `.env`.** It contains secrets. `.env.example` only has safe placeholder values.

## 3. Local Development Setup

```bash
git clone https://github.com/Abinsinguza/beehive-app.git
cd beehive-app

composer install
nvm install 22 && nvm use 22
npm install

cp .env.example .env
php artisan key:generate
# edit .env with your DB credentials

npm run build
php artisan migrate
php artisan serve
```

Visit `http://localhost:8000`.

For frontend hot-reload during development, run `npm run dev` in a second terminal instead of `npm run build`.

> **Database safety:** if pointing at a shared/production database, never run `migrate:fresh` or `migrate:reset` — these drop tables. Only plain `migrate` is safe (it only applies new migrations).

## 4. Docker Deployment (recommended for self-hosting)

This repo includes a production-ready multi-stage `Dockerfile` + `docker-compose.yml` — Nginx + PHP-FPM + your app, all in one container, managed by Supervisor.

### 4.1 Install Docker Compose (if not already present)

```bash
docker compose version
```

If that fails with `unknown command`, install the plugin:

```bash
mkdir -p ~/.docker/cli-plugins
curl -SL https://github.com/docker/compose/releases/latest/download/docker-compose-linux-x86_64 \
  -o ~/.docker/cli-plugins/docker-compose
chmod +x ~/.docker/cli-plugins/docker-compose
```

To run `docker` without `sudo` every time, add yourself to the `docker` group:

```bash
sudo usermod -aG docker $USER
newgrp docker
```

### 4.2 Build the image

```bash
docker compose build
```

This runs in 3 stages:
1. Installs PHP dependencies (`composer install`)
2. Installs PHP CLI + builds frontend assets with Vite (Wayfinder's route-type generation needs PHP available at build time)
3. Assembles the final runtime image (Nginx + PHP-FPM + your built app)

### 4.3 Set environment variables and start

```bash
export APP_KEY=$(php artisan key:generate --show)
export DB_HOST=<your-db-host>
export DB_PORT=<your-db-port>
export DB_DATABASE=<your-db-name>
export DB_USERNAME=<your-db-user>
export DB_PASSWORD=<your-db-password>

docker compose up -d
```

Visit `http://localhost:8080`.

### 4.4 Database safety during container start

By default `RUN_MIGRATIONS` is **`false`** in `docker-compose.yml` — the container will **not** touch your database automatically. To check what migrations would run, without applying them:

```bash
docker compose run --rm app php artisan migrate:status
```

Only set `RUN_MIGRATIONS=true` (or run `docker compose exec app php artisan migrate` manually) once you've confirmed it's safe for your target database.

### 4.5 If deploying behind HTTPS (real domain + reverse proxy)

Add `FORCE_HTTPS=true` to the container's environment so Laravel generates `https://` URLs for assets. Without a reverse proxy actually terminating TLS, leave it unset.

### 4.6 Useful commands

```bash
docker compose logs -f          # tail logs
docker compose ps               # check container status
docker compose down              # stop and remove containers
docker compose build --no-cache  # full rebuild after Dockerfile changes
```

## 5. Generic Linux Server (VPS, no Docker)

```bash
# System packages
sudo apt update
sudo apt install -y php8.3 php8.3-fpm php8.3-pgsql php8.3-mbstring php8.3-xml \
  php8.3-bcmath php8.3-curl php8.3-zip nginx postgresql-client unzip git
curl -sL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt install -y nodejs
curl -sS https://getcomposer.org/installer | php
sudo mv composer.phar /usr/local/bin/composer

# App setup
git clone https://github.com/Abinsinguza/beehive-app.git /var/www/beehive-app
cd /var/www/beehive-app
composer install --no-dev --optimize-autoloader
npm install && npm run build
cp .env.example .env
php artisan key:generate

# Edit .env with real production values, then:
php artisan migrate --force
php artisan config:cache
php artisan route:cache
php artisan view:cache

sudo chown -R www-data:www-data /var/www/beehive-app
sudo chmod -R 775 storage bootstrap/cache
```

Nginx server block (`/etc/nginx/sites-available/beehive-app`):

```nginx
server {
    listen 80;
    server_name your-domain.com;
    root /var/www/beehive-app/public;
    index index.php;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        fastcgi_pass unix:/run/php/php8.3-fpm.sock;
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        include fastcgi_params;
    }

    location ~ /\.(?!well-known).* {
        deny all;
    }
}
```

```bash
sudo ln -s /etc/nginx/sites-available/beehive-app /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx

# HTTPS
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

Set `FORCE_HTTPS=true` in `.env` once Certbot is configured.

## 6. Post-Deploy Checklist (any method)

- [ ] `APP_KEY` is set and unique per environment (never reuse across environments)
- [ ] `APP_DEBUG=false` in production — never expose stack traces publicly
- [ ] `FORCE_HTTPS` matches whether TLS is actually terminated somewhere in front of the app
- [ ] Database credentials point to the intended database (verify with `php artisan tinker --execute="echo DB::connection()->getDatabaseName();"`)
- [ ] Run `php artisan migrate:status` before ever running `migrate` against a shared database
- [ ] Confirm the site loads over the expected protocol with no mixed-content / SSL errors in the browser console
